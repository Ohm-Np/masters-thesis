Introduction

In this study, we are going to check how the processing time will differ - (1) sequential approach (2) parallel approach.

Four raster objects with varying resolutions were generated across the bbox extent over which the fishnets were created to carry out this study. The details of the rasters thus created follows:

  1. 0.1 degree (~11.1 km) = size 61.6 KB
  2. 0.01 degree (~1.11 km) = size 6 MB
  3. 0.001 degree (~111 m) = size 594.3 MB
  4. 0.0005 degree (~55.5 m) = size 2.3 GB

Three different functions were used to compute these rasters over 10 different fishnet grid cells: a) terra zonal b) terra extract c) exact extract

At first, the rasters were processed sequentially i.e. which uses only 1 core of the machine for the processing. Then the number of cores was increased to 8 and thus processing the rasters parallelly. Finally, the cores were increased up to 16. Thus, there are several results generated through this study which we will see in the bar chart below.

most suitable combination?

We only saw above how the three functions perform over the varying raster resolutions and varying study area polygons. So, now is the time to make recommendations for the ideal combination of functions and methods (sequential or parallelization). To carry out this analysis, all the results are aggregated and group by raster resolutions. Thus, for specific raster resolution, we could recommend what could be the most suitable combination.
